# Inbox

Put new candidate skills, agent prompts, MCP servers, and GitHub repo notes here before promoting them.

Use this lifecycle:

1. `inbox/` - not reviewed yet
2. `skills-library/experimental/` or `agents-library/experimental/` - reviewed but not trusted for real projects
3. `stable/` - tested and safe
4. `archived/` or `rejected/` - not used

Do not copy unreviewed files from inbox into a real project.
